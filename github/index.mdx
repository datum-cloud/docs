GitHub is a **core repository for work across all teams** at Datum — not just engineering. We use it for code, planning, product discussions, changelog posts, and cross-team communication.

## Organization

All source code and infrastructure lives under the [datum-cloud](https://github.com/datum-cloud) GitHub organization.

## Key Repos

| Repo | Purpose |
|---|---|
| [datum-cloud/docs](https://github.com/datum-cloud/docs) | Public docs (`main` branch) and this internal wiki (`wiki` branch) |
| [datum-cloud/datum](https://github.com/datum-cloud/datum) | Core open-source platform |
| — | — |

## Branches (this repo)

| Branch | Purpose |
|---|---|
| `main` | Public docs deployed to datum.net/docs |
| `wiki` | Internal wiki (this site) |

## Open by Default

We deliberately move **strategic and product conversations from private channels into public GitHub Discussions**. This includes team members, community participants, and customers in shaping our direction.

If a discussion could be public, it should be public. See [Values & Culture](/company/values) for more on our open-by-default approach.

## GitHub Discussions

[datum-cloud/discussions](https://github.com/orgs/datum-cloud/discussions) is where we work openly with the community — product feedback, feature requests, and roadmap conversations happen there.

## Issue Tracking

See [Issue Workflow](/github/issue-workflow) for how to link GitHub issues to wiki pages and keep project and customer documentation in sync.

## Access

All team members should have access to the `datum-cloud` org. Request access in `#engineering` on Slack if you're missing it.
