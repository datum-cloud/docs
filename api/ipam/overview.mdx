Datum Cloud manages IP address space through the `ipam.miloapis.com` API. You declare an allocatable pool from a CIDR (or carve a child pool from a parent), submit a claim requesting a sub-prefix of a given size and family, and the system records the resulting allocation carved out of the pool. Pools, claims, and allocations are resources you create and inspect with `datumctl apply -f` and `datumctl get`.

<Warning>
  The IPAM API is **alpha** (`v1alpha1`). Fields and behavior may change in backward-incompatible ways between releases.
</Warning>

## How the pieces fit together

- An **IPPool** defines allocatable address space. A root pool declares a CIDR directly, while a child pool carves a sub-prefix out of a parent pool.
- An **IPClaim** requests a sub-prefix by IP family (`spec.ipFamily`) and size (`spec.prefixLength`), targeting a pool by reference (`spec.poolRef`) or selector (`spec.poolSelector`).
- An **IPAllocation** records the specific CIDR carved out of an **IPPool** to satisfy an **IPClaim**.

<Tip>
  Inspect the exact fields for any resource with `datumctl explain`, for example `datumctl explain ippools.spec` or `datumctl explain ipclaims.spec`.
</Tip>
